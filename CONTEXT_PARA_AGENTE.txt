REGRAS DE NEGÓCIO — BÔNUS Q4 2025
================================================================================

POSIÇÕES
--------
- DIRETOR: TCV(20%) + Receita(30%) + MB%(10%) + MC%(40%) — calc_bonus_diretor()
- AE:      Receita(60%) + MB%(40%) — calc_bonus_ae()
- AE_GM:   TCV Área(20%) + Receita(30%) + MB%(50%) — calc_bonus_ae() com peso_tcv=0.2
  → AE_GM são os AEs do vertical Grupo Mult: Edmilson, Guilherme, DANILO DE SOUZA MACIEIRA
  → TCV é do vertical inteiro "Grupo Mult" (não por cliente individual)
- HUNTER:  TCV(70%) + Receita(20%) + MB%(10%)
- ESTRATEGISTAS: Receita(60%) + MB%(40%)

FÓRMULA GERAL (AE/AE_GM)
-------------------------
Bônus = Σ por WS [ sal × peso_rec × peso_ws × ating_rec × lb_gate_ws
                  + sal × peso_mb  × peso_ws × ating_mb  × mb_gate × lb_gate_ws ]
       + sal × peso_tcv × ating_tcv_area   (apenas AE_GM)

WORK STREAMS (WS) — pesos padrão Q4
-------------------------------------
apps: 70%, cloud: 8%, dados: 8%, hyper: 8%, demais: 6%
Danilo (AE_GM) tem pesos customizados: apps=72%, cloud=10%, dados=0%, hyper=10%, demais=8%
(definido em premissas_pesos_ws_pessoa.csv)

TRIGGERS Q4
-----------
- Receita/TCV: 85% da meta → abaixo = 0, entre trigger e meta = 50%→100% linear
- MB%:         meta - 1.5pp → abaixo = 0, linear até meta
- Apps: gate MB = bloqueia bônus MB apps se ating_mb = 0

TRIGGERS Q3
-----------
- Receita/TCV: 90% da meta
- MB%:         meta - 1.0pp

GATE LB POR WS (implementado recentemente)
-------------------------------------------
- Para cada WS, verifica se LB realizado >= trigger LB proporcional
- realized_lb_gate_ws: acumula APENAS clientes com budget LB > 0 naquela WS
  (evita que clientes sem budget inflem o gate)
- trigger_lb_ws_k = bgt_lb_ws * (trigger_lb_anual / meta_lb_anual)
- Se trigger = 0 → gate auto-passa

BENCHMARKS MB% POR WS
----------------------
Cloud: 34%, Dados: 35%, Hyper: 35%, Demais: 37%
Apps: calculado de verdade (custo real ou benchmark 35% se sem custo)

================================================================================
ENGINE — apuracao_engine.py
================================================================================

FUNÇÕES PRINCIPAIS
------------------
calc_bonus_ae(nome: str) → dict
  → Calcula bônus Q4 para AE, AE_GM, Hunter
  → Usa coluna q4 do budget, ae_q4, Sal_Q4

calc_bonus_ae_q3(nome: str) → dict
  → Calcula bônus Q3 para AE_GM (adicionado recentemente)
  → Usa coluna q3 do budget, ae_q3, Sal_Q3
  → _load_q3_realized() carrega SAP Q3 (vazio por ora)

calc_bonus_diretor(nome: str) → dict
  → Calcula bônus Q4 para Diretores

_load_all() → dict (cached com lru_cache)
  → Carrega TODOS os dados Q4 (budget, SAP, lookups, etc.)

_load_q3_realized() → dict (cached com lru_cache)
  → Carrega dados SAP Q3 (vazio até dados serem adicionados)

CONSTANTES IMPORTANTES
----------------------
Q4_PERIODOS = ["2025-10", "2025-11", "2025-12"]

TRIGGER_REC_Q3 = 0.90

MB_TRIGGER_DELTA_Q3 = 0.010  (Q3: 1.0pp)

RECALCULO LOCAL (FRONTEND)
---------------------------
O frontend recalcula alguns bônus localmente para exibir na tabela de WS.
Aplica lb_gate_ws ao recalcular:
  const lbGate = w.lb_gate_ws ?? 1;
  const bRec = sal * pRec * w.peso_ws * (w.ating_rec ?? 0) * lbGate;
  const gate = w.aplica_gate_mb ? (w.mb_gate ?? 1) : 1;
  const bMb  = sal * pMb  * w.peso_ws * (w.ating_mb ?? 0) * gate * lbGate;
